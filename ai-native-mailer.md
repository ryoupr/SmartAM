# AIネイティブメーラー 要件定義書

**作成日**: 2026-04-09  
**ステータス**: Draft

---

## 1. プロジェクト概要

生成AIをネイティブに統合したデスクトップメーラー。  
既存メーラーの課題（モダンでないUI・AI統合の弱さ）を解決し、メール処理をAIで自動化・効率化する。

---

## 2. ターゲット

- 個人ユーザー（macOS）

---

## 3. 解決する課題

| 課題 | 解決策 |
|------|--------|
| 既存メーラーのUIが古い | モダンなデザインシステムを採用 |
| AI統合が弱い・後付け感がある | AIをコア機能として設計段階から組み込む |
| 複数LLMプロバイダーの切り替えが面倒 | LiteLLM経由で統一インターフェース |

---

## 5. 全機能要件

### 5.1 基本メール機能

- [ ] IMAP/SMTP対応（Gmail / Outlook / 汎用）
- [ ] 複数アカウント管理（アカウント切り替え・統合受信トレイ）
- 新規作成時の送信元: サイドバーで選択中のアカウントがデフォルト、作成画面のFromドロップダウンで切替可能
- [ ] メール一覧・詳細表示
- [ ] 送受信・返信・転送・削除
- [ ] CC/BCC
- [ ] 添付ファイル（送受信・プレビュー・ダウンロード）
- [ ] スレッド表示
- [ ] メール同期（IMAP IDLE + ポーリング併用）
  - IMAP IDLE: 常時接続で新着を即時受信（対応サーバーのみ）
  - ポーリング: 5分間隔でバックグラウンド同期（IDLE非対応・接続断時のフォールバック）
  - 手動トリガー: 受信トレイを開く・アカウント切替時に対象アカウントの新着を即時確認
- [ ] アーカイブ
- [ ] スター付け
- ボタン表示仕様:
  - 詳細ペイン: アイコンのみ表示（ツールチップで機能名を補足）
    - 返信「↩」転送「→」アーカイブ「📦」削除「🗑」スター「⭐」カレンダー「📅」要約「📝」下書き「✍」翻訳「🌐」
  - 作成系画面（新規作成・返信・転送）: アクションボタンはテキストラベル付き
    - 送信「📤 送信」= 緑（#a6e3a1）、添付「📎 添付」キャンセル「キャンセル」= グレー
    - Fromフィールドのテキスト色: デフォルトアカウント = 緑（#a6e3a1）
- AI機能ボタンのカラーコーディング（アクティブ時の色）:
  - 📝 要約 = 緑（#a6e3a1）
  - ✍ 下書き = 黄（#f9e2af）
  - 🌐 翻訳 = 青（#89b4fa）
  - 📅 カレンダー = 青（#89b4fa）
  - ⭐ スター付け = 黄（#f9e2af）
  - 🔄 再生成ボタン = 紫（#cba6f7）で全パネル共通
- ボタンの非アクティブ時スタイル:
  - AI機能ボタン（📝 ✍ 🌐 📅）: fillColor=#313244、strokeColorに各機能カラーの枠線
  - 非AI機能ボタン（↩ → 📦 🗑 ⭐）: fillColor=#313244、strokeColor=#45475a（汎用グレー枠線）
- [ ] 検索バー（メール一覧上部）
- [ ] メール同期ステータス表示（最終同期時刻 + 🔄手動更新ボタン）
- [ ] LLMプロバイダーバッジ（サイドバー下部に接続中のLLM名を表示）
- [ ] トースト通知（操作完了時、5秒で自動消去）
  - 可逆操作（アーカイブ・カレンダー登録等）: 「元に戻す」/「取り消す」リンク付き
  - 不可逆操作（メール送信）: 確認のみ（取り消し不可）
- [ ] 検索バー（メール一覧上部、フォーカス時strokeColor=#cba6f7、ハイライト色=#f9e2af）
- 署名の自動挿入: 新規作成・返信・転送の全作成系画面で挿入（Fromアカウントに紐づく署名）

### 5.2 AI機能

**設計方針: オンデマンド実行（トークン節約）**
- メール表示時にAIは自動実行しない
- ユーザーがボタンを押下したときのみAPIコールを発行する
- 各AI機能は独立したボタンとして詳細ペインに配置
- AI処理中: 押下されたボタンをopacity=60%で無効化表示、パネル内にローディングUI（全AI機能共通）

| 機能 | トリガー | 優先度 |
|------|---------|--------|
| 要約 | 📝 ボタン押下 | 高（MVP） |
| 返信下書き | ✍ ボタン押下 → 2ステップ生成 | 高（MVP） |
| 翻訳 | 🌐 ボタン押下 | 高（MVP） |
| カレンダー登録 | 📅 ボタン押下 | 高（MVP） |
| 優先度・カテゴリ自動分類 | 🏷 ボタン押下 | 中（Phase 2） |
| アクションアイテム抽出 → Jira登録 | ⚡ ボタン押下 | 中（Phase 2） |
| Claude Code統合 | ローカルのClaude Codeとの連携 | 低（Phase 4） |

**返信下書きフロー（2ステップ）:**
1. ✍ボタン押下 → AIがメール内容を分析し、最大5個の返答ニュアンス（承諾・拒否・保留等）を提案
2. ユーザーがニュアンスを選択 → 選択されたニュアンスで返信案を生成（Step2のUIは全ニュアンス共通）

**ニュアンス選択肢のカラーコーディング:**
- ✅ 承諾する = 緑（#a6e3a1）
- ❌ 辞退する = 赤（#f38ba8）
- ⏳ 保留する = 黄（#f9e2af）
- ❓ 質問する = 青（#89b4fa）
- 🔄 委任する = グレー（#a6adc8）

**翻訳対応言語:**
- English, 中文（簡体）, 中文（繁體）, 한국어, Français, Deutsch, Español, Português, Русский, العربية

### 5.5 キーボードショートカット

**設計方針:**
- Gmail のショートカットをベースに、SmartAM 固有のAI機能キーを追加
- 単一キー押下で動作（修飾キー不要）。作成モーダル・設定画面が開いている間は無効化
- `?` でショートカット一覧ヘルプを表示
- 将来的にカスタマイズ可能にする（Phase 2）

#### カスタマイズUI（設定画面内）

- 設定画面のグローバルタブに「キーボードショートカット」を追加
- アクション名とキーバインドを一覧表示。キー欄をクリックすると編集モード（`⌨ キーを押してください...`）に入り、押下されたキーを記録
- キー重複時は警告を表示（`⚠ 「x」は既に「○○」に割り当てられています`）し、保存をブロック
- 「デフォルトに戻す」ボタンで全ショートカットを初期値にリセット
- カスタムキーバインドは `settings.json` の `shortcuts` フィールドに保存

#### ナビゲーション

| キー | アクション | Gmail互換 |
|------|-----------|-----------|
| `j` | メール一覧: 次のメールへ移動 | ✅ |
| `k` | メール一覧: 前のメールへ移動 | ✅ |
| `Enter` / `o` | 選択中のメールを開く | ✅ |
| `u` | メール詳細を閉じる（一覧に戻る） | ✅ |
| `g` → `i` | 受信トレイへ移動 | ✅ |
| `g` → `s` | スター付きへ移動 | ✅ |
| `g` → `t` | 送信済みへ移動 | ✅ |
| `g` → `d` | 下書きへ移動 | ✅ |
| `g` → `a` | すべてのメールへ移動 | ✅ |

#### メール操作

| キー | アクション | Gmail互換 |
|------|-----------|-----------|
| `r` | 返信 | ✅ |
| `f` | 転送 | ✅ |
| `a` | アーカイブ | — |
| `#` | 削除（確認ダイアログ表示） | ✅ |
| `s` | スター切り替え | ✅ |
| `z` | 直前の操作を元に戻す（トースト「元に戻す」と同等） | ✅ |

#### AI機能（SmartAM固有）

| キー | アクション | 備考 |
|------|-----------|------|
| `d` | AI返信下書き（✍パネル開閉） | Gmail の `d`（新規タブ作成）を上書き |
| `t` | 翻訳（🌐パネル開閉） | translate の `t` |
| `y` | 要約（📝パネル開閉） | yōyaku の `y` |
| `l` | カレンダー登録（📅パネル開閉） | caLendar の `l` |

#### 作成・検索

| キー | アクション | Gmail互換 |
|------|-----------|-----------|
| `c` | 新規メール作成 | ✅ |
| `/` | 検索バーにフォーカス | ✅ |
| `Escape` | モーダル/パネルを閉じる | — |

#### 作成モーダル内

| キー | アクション | Gmail互換 |
|------|-----------|-----------|
| `⌘+Enter` | 送信 | ✅ |
| `Escape` | 作成をキャンセル | — |

#### 実装仕様

- **イベントハンドリング**: `+page.svelte` の `<svelte:window on:keydown>` でグローバルにキャプチャ
- **無効化条件**: `composeMode !== null` または `showSettings === true` または `document.activeElement` が `input`/`textarea`/`select` の場合はショートカットを無効化
- **`g` → X シーケンス**: `g` 押下後 1秒以内に次のキーが押されなければリセット。`g` 押下中は画面下部に「g → ?」のヒントを一時表示
- **ヘルプ表示**: `?` 押下でショートカット一覧をモーダル表示（閉じるは `Escape` または再度 `?`）

## 6. 技術スタック

### 6.1 フレームワーク: Tauri v2

- Rustバックエンド + macOS WebKit（WKWebView）
- Electronと比較してメモリ約58%削減、バンドルサイズ約96%削減
- LiteLLMをsidecarバイナリとしてバンドル（`externalBin` + `tauri_plugin_shell`）

**フロントエンド**: SvelteKit  
**バックエンド**: Rust（Tauri core）

### 6.2 LLM統合: LiteLLM Proxy（sidecar）

| プロバイダー | モデル例 |
|-------------|---------|
| Ollama（ローカル） | llama3, mistral, etc. |
| OpenAI | gpt-4o, gpt-4o-mini |
| Anthropic | claude-3-5-sonnet |
| AWS Bedrock | claude on bedrock |
| Google Gemini | gemini-2.5-flash |

### 6.3 メール処理

- Rust: `async-imap`（IMAP）、`lettre`（SMTP）

### 6.4 Claude Code統合（Phase 4）

- `ANTHROPIC_BASE_URL=http://localhost:11434` でOllamaにリダイレクト可能
- メーラーからこの環境変数を制御する形で統合
